null x undefined

Dados primitivos:

  • string
  • number
  • undefined
  • null
  • bollean
  • symbol


null --> Dizemos explicitamente que tal variável não deve apontar para nenhum endereço de memória.
A variável não tem valor e não poderemos definir valor posteriormente

undefined --> Quando não definimos um valor para a variável, entretanto podemos definir o valor posteriormente se necessário.
O JavaScript define que tal variável não aponta para nenhum endereço de memória


Dados por referência:

Apontam para o mesmo local/endereço de memória,
ou seja, se alteramos o valor de um dado afeta o outro

Já com dados primitivos isso não acontece,
se alteramos um dado o outro não é afetado
